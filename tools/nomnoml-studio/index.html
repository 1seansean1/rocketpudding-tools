EXAMPLE

DIRECTIVES

One per line, before the diagram. Syntax: #name: value. Defaults shown are nomnoml 1.7 built-ins.

#direction
Layout flow: down or right.
default: down
#spacing
Distance between nodes and ranks, in px.
default: 40
#padding
Space between a node's border and its content, in px.
default: 8
#fill
Node fill colors, semicolon-separated, cycled by nesting depth: #fill: #eee8d5; #fdf6e3.
default: #eee8d5; #fdf6e3
#stroke
Color of borders, edges, arrowheads, and text.
default: #33322E
#font
Font family for all diagram text. Use #font: IBM Plex Sans to match this site.
default: Helvetica
#fontSize
Text size in px.
default: 12
#lineWidth
Stroke width of borders and edges.
default: 3
#background
Canvas background color.
default: transparent
#edges
Edge corner style: rounded or hard.
default: rounded
#arrowSize
Arrowhead scale factor.
default: 1
#bendSize
Corner radius factor for rounded edges.
default: 0.3
#gutter
Margin around the whole diagram, in px.
default: 20
#edgeMargin
Gap between node border and edge endpoint, in px.
default: 0
#leading
Line-height multiplier for node text.
default: 1.35
#zoom
Render scale factor.
default: 1
#title
Diagram title; this app also uses it as the download filename.
default: (empty)
#ranker
Layering algorithm: network-simplex, tight-tree, or longest-path.
default: network-simplex
#acyclicer
Set to greedy to reverse some edges so cyclic graphs lay out cleanly.
default: (off)
#gravity
Cluster gravity (experimental).
default: 1

CUSTOM STYLES

#.warn: fill=#f7e9e7 stroke=#b3403a dashed defines a classifier, used as [<warn>Node]. Modifiers: fill=, stroke=, align=center, direction=, visual= (class, note, database, actor, ...), dashed, bold, italic, underline.

NODES

[name] box. [name|attributes|methods] compartments; ; is a line break inside a compartment. Nesting: [outer|[inner]->[inner2]].

Built-in classifiers: <abstract> <instance> <reference> <note> <package> <frame> <database> <actor> <usecase> <start> <end> <state> <choice> <sync> <input> <table> <label> <hidden>

EDGES

- line · -> arrow · <-> both ways · --> dashed arrow · -:> generalization · --:> dashed generalization · o-> aggregation · +-> composition · -/- hidden.

Edge labels and cardinality: [A] label -> [B], [A] 1 - 0..* [B].
