JSONPath cheat sheet

Roots & steps

$the root of the document — every path starts here
@the current node — used inside filter expressions
.namechild by name (dot notation), e.g. $.store
['name']child by name (bracket notation) — use for keys with spaces or dots
..recursive descent — scan self and all descendants, e.g. $..price
*wildcard — every child (of an object or array), e.g. $.store.*

Array indexing

[n]element at index n (0-based), e.g. [2]
[-n]counts from the end — [-1] is the last element
[a,b]union — several indices or names at once, e.g. [0,2]
[start:end]slice — from start up to (not including) end, e.g. [0:2]
[:n]first n elements; [n:] from n onward; [-2:] last two
[start:end:step]slice with a step, e.g. [::2] every other

Filters  [?(…)]

[?(@.x)]items where property x exists / is truthy
[?(@.x>1)]comparisons: == != < <= > >=
@.x=="y"string compare (quote the literal)
&&  ||combine conditions, e.g. @.a>1&&@.b<9
@.arr.lengthlength of an array/string property
@.price<@root.maxreference the document root from inside a filter (use @root, not $)
[?(/re/.test(@.s))]regex test on a value

Result forms & tips

valuesthis tool’s left toggle — the matched values as pretty JSON
pathsthe right toggle — each match’s normalized path, e.g. $['store']['book'][0]
$..*every value in the document (deep)
$.a.b.ca missing step yields no matches, not an error
~property name of a match (rather than its value)
^the parent of a match
