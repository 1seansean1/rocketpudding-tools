DAG LAYOUT LAB

FAQ & Reference

A sandbox for laying out a nested, grouped directed graph — a system of containers → subsystems → leaf nodes — while experimenting with layout algorithms, edge routing, and grid discipline. This page documents what each part does, and is candid about what is still a placeholder or stub.

functional partial / illustrative stub / placeholder

Layout functions

The DETAILS panel's algorithm picker drives placement. The default is ★ Rank-weighted stress — weighted stress seeded by node rank, with group cohesion, crossing reduction, and wire discipline (the routing style feeds back into placement, so a right-angle style yields a circuit-board grid).

Also available: the stress family (SMACOF, Kamada–Kawai, Pivot MDS), Layered / Sugiyama, Tree, Bipartite, Swim-lane Bands, Grid, Coffman–Graham, Orthogonal, Radial, force-directed variants (Fruchterman–Reingold, ForceAtlas2, Eades, GEM, Yifan Hu), Spectral, and diagrammatic forms (circular, concentric, spiral, hive, arc, quadrant, treemap, clustered).

The PARAMETERS sliders (rank weight, ideal edge length, crossing penalty, evenness, wire discipline, group cohesion, repulsion, gravity, spacing, iterations…) tune the active algorithm; sliders it doesn't read are dimmed. Switching algorithms re-runs the layout and animates the change. The ⚡ benchmark overlay runs every algorithm and ranks them by readability.

Grid snapping

  • Everything lives on a 26px grid. Node dimensions are exact 26px multiples, so a snapped box tiles on both edges — not just one corner.
  • Layout output, node drags, and container drags/resizes all settle onto the grid on release; a node's top-left corner lands on the lattice.
  • Container frames expand out to the nearest gridlines so nested boxes align.
  • Resizing scales children smoothly while dragging, then snaps on release (no cell-to-cell jumping); the NW corner stays put.
  • Tradeoff: quantising sizes to whole cells makes this graph's nodes fall on one size tier, so boxes read uniform.

Edge routing

EDGE ROUTING panel
  • Geometry: Direct, Curved, Bézier, Spline, Orthogonal, Sloppy. Corners: sharp / rounded (+ radius). Terminal: perpendicular / direct.
  • Attachment: port policy (Nearest / Spread / Center), side margin, port gap — where edges dock on a node's sides.
  • Sliders bundle margin, port gap, clearance, radius, curve tension, stroke width.
  • Style: stroke colour, dash, arrowheads (source / target), and bridge hops — the more-horizontal edge arcs over the one it crosses. Container-to-container links render as dashed boundary edges.
  • Edges have a fat invisible hit-band, so they're easy to click even though they're drawn thin.

Colour & properties

  • Auto colour (droplet toggle in the toolbar): each innermost subsystem gets its own hue, with its nodes tinted as a gradient from it; wrapper containers stay neutral. Toggle off for plain slate.
  • Select a container to open PROPERTIES — override its border colour/style/width and fill colour/opacity, and rename its label. These overrides persist.
  • Double-click a leaf node to rename it inline; double-click a container's label to edit it.
  • Node text (tag / title / metadata / port) and container labels share one uppercase mono treatment; the top of every box reserves a label margin.

Click, hover & drag behavior

  • Hover a node/edge to focus it: its layer stays bright while other layers subdue (leaf nodes dim less than containers & edges, preserving depth). Hovering a node shows its URI bottom-right and highlights its file in the tree.
  • Click to pin that focus; it holds until you click empty canvas.
  • Drag a node to move it (grid-snapped); grab a container by its north edge (the ≡ handle) to move the whole subtree, or the SE grip to resize.
  • Pushing toggle: dragging shoves same-level neighbours aside with a margin; the cascade reflows sibling containers and their parents. Resizing then shrinks a container only until its parts are packed.
  • Ctrl/⌘-click multiple nodes, then right-click for a menu to Group / Ungroup.
  • Scroll to zoom, drag empty canvas to pan.

Left columns & the URI

  • The work tree is generated from the live graph: containers become folders, leaf nodes become files (title.ts). It's bidirectional — hovering a node highlights & auto-scrolls to its file row; clicking a file row selects and centers its node.
  • Both columns collapse/expand from their headers, or together via the toolbar's two-panel button.
  • Each node carries a content-addressed Caramel URI: repo@commit#path:symbol:line — repo = outermost container, path = the subsystem chain + file, commit = a short hash. Shown bottom-right on hover.

Shortcuts & persistence

  • ⌘/Ctrl + F opens graph search — matches tags, titles, ports, metadata, and lights the results like a hover. Esc clears search / selection.
  • Your layout, manual positions, algorithm, routing settings, container styles/labels and tags persist across reloads (saved locally). Grouping/ungrouping and node renames are in-session and reset on a hard reload.
  • Tag any node or container from its Properties panel to organize and find it later.
  • The status line up top reports node / edge / group counts for the current system.
  • Layout changes animate so you can see how an algorithm reorganizes; turn motion off for instant snaps.
