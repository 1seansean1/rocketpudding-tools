HOW IT WORKS
Import extraction is regex-based (fast, approximate): JS/TS import … from '…', import '…', export … from '…', require('…'), import('…'); Python import x, from x import y (incl. leading-dot relatives). Relative specs resolve against archive paths, trying exact match, .js/.jsx/.ts/.tsx/.mjs/.cjs/.py extensions, .js→.ts swap, index.*, and for Python mod.py / mod/__init__.py from root or the importing file's directory. Unresolved or package imports become dimmed external pills. Imports built from variables or template strings, and path aliases like @/x, are not resolved. Layout ranks by longest path over the cycle-condensed DAG; cycles are found with Tarjan's SCC and drawn red.
