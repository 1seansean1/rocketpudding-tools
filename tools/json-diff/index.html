How this diff works

    
Structural, not textual. Both documents are parsed and their trees are walked. Whitespace and object key order never register as a change — only the data does.

    
Array matching. Index-wise compares position by position. Keyed / LCS matches elements by the chosen id key (falling back to a value-based longest-common-subsequence), so an insert or reorder is one add, not a cascade of changes down the tail.

    
Options. ignore array order matches array elements as a multiset (a pure reorder becomes zero changes). treat null as absent makes "k": null equivalent to a missing key. num tol treats two numbers within the tolerance as equal.

    
RFC 6902 export. The JSON-Patch is an ordered add/remove/replace script generated with correct running array indices. Applied to the Left document it reconstructs the Right document — the badge verifies this live.
